# Quick Start Guide

This guide will help you get up and running with the Agent Orchestration System in under 5 minutes.

## Step 1: Prerequisites

Ensure you have:
- .NET 10 SDK installed
- An API key for either OpenAI (ChatGPT) or Anthropic (Claude)

## Step 2: Clone and Build

```bash
git clone https://github.com/shibubh/alpha-agent-app.git
cd alpha-agent-app
dotnet build AgentOrchestration.sln
```

## Step 3: Configure Your API Key

### Using Environment Variables (Recommended)

For ChatGPT:
```bash
export OPENAI_API_KEY="sk-your-key-here"
```

For Claude:
```bash
export CLAUDE_API_KEY="sk-ant-your-key-here"
```

### Or Edit the Configuration File

1. Navigate to `src/AgentOrchestration.CLI/`
2. Open `appsettings.json`
3. Add your API key:

```json
{
  "AIProvider": {
    "Provider": "ChatGPT",
    "ChatGPT": {
      "ApiKey": "sk-your-key-here",
      "Model": "gpt-4"
    }
  }
}
```

## Step 4: Run the Application

### Option A: Using the Run Script (Linux/macOS)
```bash
./run.sh
```

### Option B: Manual Execution
```bash
cd src/AgentOrchestration.CLI
dotnet run
```

### Option C: From Solution Root
```bash
dotnet run --project src/AgentOrchestration.CLI/AgentOrchestration.CLI.csproj
```

## Switching Between AI Providers

To switch from ChatGPT to Claude or vice versa:

1. Edit `src/AgentOrchestration.CLI/appsettings.json`
2. Change the `Provider` field to either `"ChatGPT"` or `"Claude"`
3. Ensure the corresponding API key is configured

## Troubleshooting

### "API key not configured" error
- Make sure your API key is set either in `appsettings.json` or as an environment variable
- Double-check the environment variable name: `OPENAI_API_KEY` or `CLAUDE_API_KEY`

### Configuration file not found
- Make sure you're running the application from the correct directory
- The `appsettings.json` file must be in the same directory as the executable

### Build errors
- Ensure .NET 10 SDK is installed: `dotnet --version`
- Clean and rebuild: `dotnet clean && dotnet build`
